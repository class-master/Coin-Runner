# Day4：演出と仕上げの日（7人×3時間用シナリオ）

## 0. 今日のゴール（先生が最初に読む用）

3時間で、次を「クラス全体」として完成させます。

1. Title → Play → GameOver の流れが **一通りプレイして気持ちよい状態** になっている  
2. 難易度カーブやパラメータを調整し、**「遊びやすさ」と「やりごたえ」のバランス** を取る  
3. 必要に応じて **簡単な演出（色変更／テキスト演出／SE/BGMのフックだけ）** を追加する  
4. Day1〜Day3で入れた機能を振り返り、「どこを伸ばしたいか」を言葉にできる

※ Day4は「新機能大量追加」ではなく、「既存機能の磨き込みと振り返り」がテーマです。

## 3. 各役割の「やることチェックリスト」

### 3-1. A：Title画面＆導線担当

- [ ] `scenes/title.py` のラベル文言を、実際のゲーム内容に合うように調整した
- [ ] Enter/Space/タップでPlayへ遷移する処理を完成させた
- [ ] 余裕があれば、タイトルに簡単なサブタイトルやバージョン表記を追加した

### 3-2. B：GameOver画面の演出担当

- [ ] GameOver時に表示するメッセージやボタン配置を設計した
- [ ] Retry時に何がリセットされるかを確認し、必要ならメッセージで説明した
- [ ] ExitでTitleに戻ったときの印象が暗くなりすぎないように工夫した

### 3-3. C：難易度パラメータ調整担当

- [ ] difficultyの `step_sec` / `increment` / `max_stage` の意味を再確認した
- [ ] 実際に何度かプレイし、「ちょうどよい速度上昇」を探した
- [ ] 最終的に採用する値と、その理由をコメント／Day4バグログに書いた

### 3-4. D：コイン＆障害物パラメータ担当

- [ ] Day2で調整した gap関連パラメータを見直し、Day3の難易度と合わせて再調整した
- [ ] 「序盤は優しめ、中盤以降から少しシビア」など、設計方針を言葉にしてから値を決めた
- [ ] 調整結果をGに伝え、テストプレイで感想をもらった

### 3-5. E：HUD＆メッセージ演出担当

- [ ] Score/Speed/Lv/Energy などの表示順や強調具合を整えた
- [ ] デバッグ情報と通常情報の切り替えを分かりやすくした（必要ならF1切替）
- [ ] GameOverやTitleとの一貫性を持たせるように文言をそろえた

### 3-6. F：サウンド／拡張フック担当

- [ ] Kivyでサウンドを鳴らすにはどうするか調査した（`SoundLoader` など）
- [ ] すぐには実装しなくてもよいので、「ここにBGM」「ここでSE」といった場所をコメントでマークした
- [ ] 余裕があれば、1種類だけでもSEを追加してみた（ジャンプ時／GameOver時など）

### 3-7. G：バグログ＆ふりかえり担当（`docs/Day4_buglog.md`）

- [ ] Day1〜Day4を通して気になっていたバグを洗い出し、未解決のものをリストアップした
- [ ] 今日直したもの／次の機会に回すものを分けて整理した
- [ ] 「この4日間でできるようになったこと」を自分の言葉で1つ以上書いた

- 「とりあえず派手な演出」を足しすぎて、プレイしづらくしない
- 大きな設計変更をDay4で突然行わない（時間内に終わる範囲で）
- 自分の担当だけでなく、全員でゲーム全体をプレイしてフィードバックを出す
